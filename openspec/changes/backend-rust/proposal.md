# 提案：后端从 Node.js 迁移到 Rust

## 为什么做这个变更？

### 性能优势
- **并发性能**：Rust 的 async/await 和零成本抽象，并发性能远超 Node.js
- **内存安全**：编译时保证内存安全，无 GC 停顿
- **性能提升**：预计 API 响应速度提升 5-10 倍

### 技术优势
- **类型安全**：强类型系统，编译时捕获错误
- **生态系统**：Actix-web/Axum 等现代 web 框架
- **可维护性**：Rust 代码更易维护和重构

## 将改变什么？

### 新增
- `server-rust/` - Rust 后端项目
- 使用 Actix-web 或 Axum 框架
- Serde 序列化/反序列化
- Tokio 异步运行时

### 保留
- `server/` - Node.js 版本（暂时保留作为参考）
- OpenAPI 规范文件（兼容）
- API 接口定义（完全兼容）

### 迁移
- 所有 API 逻辑
- 错误处理机制
- 响应格式（完全兼容）

## 影响范围
- 后端服务 (server-rust/)
- 性能显著提升
- 前端无感知（API 完全兼容）
