# 提案：完善 Rust 后端功能并移除 Node.js 版本

## 为什么做这个变更？

### 业务目标
- **长期技术栈统一**：只维护一套高性能 Rust 后端
- **性能优化**：充分利用 Rust 的并发和内存优势
- **代码简化**：减少维护成本，避免两套代码

### 当前问题
1. Rust 后端功能不完整（缺少购物清单、菜谱查询）
2. 缺少 OpenAPI 文档和 Swagger UI
3. 同时存在两套代码，维护成本高

## 将改变什么？

### 新增功能
1. **购物清单 API** (`POST /api/shopping/generate`)
   - 根据菜品生成食材清单
   - 智能调整分量
   - 分类展示食材

2. **菜谱查询 API** 
   - `GET /api/recipe/:name` - 获取菜谱详情
   - `GET /api/recipe/search` - 搜索菜谱

3. **API 文档**
   - OpenAPI 规范
   - Swagger UI 界面
   - 交互式 API 测试

### 移除内容
- `server/` 目录（Node.js 后端）
- 相关的 Node.js 配置文件

### 优化内容
- 完善 AI 集成实现
- 添加缓存机制
- 性能测试和优化

## 影响范围
- 后端服务完全切换到 Rust
- API 接口保持兼容（前端无感知）
- 性能显著提升
